# AgentHive Cloud — 总体架构设计文档 v1.0

## 1. 愿景

构建一个 **AI 驱动的软件研发平台**，用户通过自然语言 Chat 界面，指挥一支 AI Agent 团队完成软件开发的全生命周期（需求分析 → 架构设计 → 编码 → 测试 → 部署）。

同时，平台包含完整的 **电商 SaaS 能力**（用户、支付、订单、购物车、物流），支撑商业化运营。

## 3. 技术栈

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 前端 | Nuxt 3 + Vue 3 + Element Plus + Tailwind | 已有基础，SSR + SPA 混合 |
| API 网关 | Spring Cloud Gateway | 统一路由、鉴权、限流、日志 |
| 注册中心 | Nacos | 阿里生态，配置+注册一体 |
| Node.js 服务 | Express + TypeScript | AI 推理链路，轻量高效 |
| Java 服务 | Spring Boot 3.2 + Spring Cloud 2023 | 企业级标准，生态完善 |
| 数据库 | PostgreSQL 16 | 主业务库 |
| 缓存 | Redis 7 | 会话、缓存、分布式锁 |
| 消息队列 | RabbitMQ | 异步解耦（订单→支付→物流） |
| 对象存储 | MinIO | 文件、日志、镜像 |
| 可观测性 | OTel + Grafana LGTM | 已实施完成 |
| AI 推理 | Ollama (qwen3:14b) | 本地部署，成本可控 |

## 4. 通信契约

### 4.1 跨服务调用
- **同步**：OpenFeign + Nacos 服务发现
- **异步**：RabbitMQ 事件驱动
- **前端 → 后端**：REST API + WebSocket (实时推送)

### 4.2 统一响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "traceId": "abc123",
  "timestamp": 1713715200000
}
```

### 4.3 错误码规范
| 范围 | 含义 |
|------|------|
| 200-299 | 成功 |
| 400-499 | 客户端错误 |
| 500-599 | 服务端错误 |
| 6000-6999 | AI Agent 业务错误 |
| 7000-7999 | 电商业务错误 |

## 5. 安全设计

- **网关层**：JWT 鉴权 + API 限流 + IP 白名单
- **服务层**：RBAC 权限控制 + 数据脱敏
- **支付层**：幂等性 + 分布式锁 + 对账机制
- **通信层**：mTLS (K8s) + 签名验证

## 6. 高并发设计

| 场景 | 策略 |
|------|------|
| 用户注册/登录 | Redis 缓存 + 布隆过滤器防重复 |
| 商品/订单查询 | 多级缓存 (Caffeine + Redis) + 读写分离 |
| 下单 | 库存预扣 (Redis Lua) + 异步落库 + 消息队列 |
| 支付 | 幂等 Token + 分布式锁 + 对账补偿 |
| AI Agent 任务 | 限流 + 队列 + 超时熔断 |

## 7. 数据一致性

- **强一致性**：支付状态变更（Seata AT 模式）
- **最终一致性**：订单→物流（消息队列 + 本地消息表）
- **BASE**：Agent 任务状态、日志流水
