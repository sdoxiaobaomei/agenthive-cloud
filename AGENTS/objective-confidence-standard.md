# Objective Confidence 评分标准

> 替代主观自评，基于可验证的客观指标计算 confidence_score。
> 版本: v1.0 | 生效日期: 2026-04-27

## 评分公式

```
confidence_score = Σ(指标得分 × 权重)
```

| 指标 | 权重 | 验证方式 | 满分条件 |
|------|------|----------|----------|
| **AC 完整实现** | 0.25 | 逐项核对 TICKET.acceptance_criteria | 全部实现 |
| **编译/类型通过** | 0.10 | `mvn clean package` / `tsc --noEmit` | 0 错误 |
| **测试覆盖** | 0.15 | 新增测试数 + 现有测试通过率 | 新增 ≥3 个，全部通过 |
| **安全扫描** | 0.10 | 无硬编码 secret、无 SQL 注入风险 | 无新增漏洞 |
| **Reflection 质量** | 0.10 | 包含架构决策 + 踩坑记录 + 测试策略 | 完整且 ≤1.5K tokens |
| **Skill 沉淀** | 0.15 | RESPONSE 中 skill_candidate=true 且有 skill_summary | 有候选 skill |
| **代码风格** | 0.10 | 符合团队规范（Lombok、结构化日志等） | 无风格违规 |
| **文档更新** | 0.05 | schema/API/部署文档同步更新 | 公共接口已文档化 |

## 评分区间与路由

| 分数 | 路由 | 说明 |
|------|------|------|
| 0.90-1.00 | 自动通过 | 极少数情况，所有指标完美 |
| 0.75-0.89 | Lead 快速审查 | 常规情况，抽查关键文件 |
| 0.60-0.74 | Lead 详细审查 | 有短板，需逐项核对 |
| < 0.60 | 打回重派 | 严重缺陷，必须修复 |

## 虚高识别规则

以下情况 **强制降分**：
- 无 skill_candidate 标记 → -0.15
- 无 reflection 或 reflection < 300 字 → -0.10
- 声称"所有测试通过"但未列出新增测试数 → -0.10
- 编译通过但存在 warning（未处理）→ -0.05
- 代码风格明显违规（如使用 @Data on @Entity）→ -0.10

## 示例

**JAVA-006（流量收益结算定时任务）**

| 指标 | 实际 | 得分 |
|------|------|------|
| AC 完整实现 | 8/8 实现 | 0.25 |
| 编译通过 | mvn 通过 | 0.10 |
| 测试覆盖 | 新增 7 个，46 个全通过 | 0.15 |
| 安全扫描 | 无新增 secret | 0.10 |
| Reflection 质量 | 完整，有编码损坏 | 0.08 |
| Skill 沉淀 | 无 skill_candidate | **0.00** |
| 代码风格 | 符合规范 | 0.10 |
| 文档更新 | schema.sql 已更新 | 0.05 |
| **小计** | | **0.83** |
| **虚高调整** | 无 skill_candidate (-0.15) | **0.68** |

**结论**: 0.68 → Lead 详细审查，打回要求补充 skill_candidate。

---

> **与旧标准的区别**: 旧标准由 Specialist 自评（通常 0.91-0.95），新标准由客观指标计算，不可人为操纵。
