## Migration Rules

1. **File naming**: `YYYYMMDDhhmmss_description.sql`
2. **Location**: `apps/api/src/db/migrations/`
3. **Tool**: `node-pg-migrate`
4. **Idempotency**: ALL migrations must be idempotent
   ```sql
   -- For columns: use IF NOT EXISTS
   ALTER TABLE table_name ADD COLUMN IF NOT EXISTS col_name TYPE;
   
   -- For constraints: use DO $$ wrapper (PG 16 compat)
   DO $$
   BEGIN
     IF NOT EXISTS (
       SELECT 1 FROM pg_constraint WHERE conname = 'constraint_name'
     ) THEN
       ALTER TABLE table_name ADD CONSTRAINT constraint_name ...;
     END IF;
   END $$;
   ```
5. **No destructive changes** in migrations on existing tables with data
6. **Test**: Use `scripts/test-migrations.js` for clean DB replay testing

---

## PG 16 Compatibility Notes

- `ADD CONSTRAINT IF NOT EXISTS` is **NOT supported** in PG 16
- Always use `DO $$ BEGIN IF NOT EXISTS ... END $$;` wrapper for constraints
- `ADD COLUMN IF NOT EXISTS` IS supported
